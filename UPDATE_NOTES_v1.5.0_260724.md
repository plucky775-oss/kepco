# v1.5.0 업데이트 내용

## 핵심 Check Point 계층 구분

원본 96페이지 PDF의 핵심 Check Point 기호를 항목별로 다시 대조해 구조화했습니다.

| 원본 기호 | 앱 구분 | 표시 방식 |
|---|---|---|
| ■ | 상위 항목 | 굵은 글씨, 진한 왼쪽 선, 강조 배경 |
| ○ | 하위 또는 독립 항목 | 원형 기호, 상위 항목이 있을 때 들여쓰기 |
| - | 세부 항목 | 대시 기호, 2단계 들여쓰기 |
| 표시 없음 | 원본 표시 없음 | 별도 기호 없이 독립 항목으로 유지 |

전체 2,578개 항목의 원본 기호는 다음과 같습니다.

- 사각형: 371개
- 원형: 2,176개
- 대시: 28개
- 원본 표시 없음: 3개

같은 작업절차 구간 안에서 사각형 뒤에 오는 원형은 해당 사각형의 하위 항목으로 연결했습니다. 다음 사각형이 나오면 하위 관계가 새 사각형으로 바뀝니다. 사각형보다 먼저 나오거나 사각형이 없는 구간의 원형은 독립 항목으로 유지했습니다.

## 작성 화면

- `■` 상위 항목과 `○` 하위 항목의 기호·굵기·들여쓰기를 구분했습니다.
- 상위 항목은 진한 왼쪽 강조선과 연한 배경으로 표시합니다.
- 하위 항목은 들여쓰기와 얇은 연결선으로 표시합니다.
- 대시 세부 항목은 하위 항목보다 한 단계 더 들여씁니다.
- 기존 `중점` 보조 배지는 제거하고 원본 기호로 직접 구분합니다.
- 확인·이상·해당 없음, 사진, 조치 입력 기능은 그대로 유지합니다.

## PDF

- 작업절차 구간 제목을 체크표 안의 구분 행으로 표시합니다.
- `■ / ○ / -` 기호와 상·하위 들여쓰기를 PDF에도 반영합니다.
- 사각형 상위 행은 굵은 글씨와 배경으로 구분합니다.
- PDF 렌더링 버전을 갱신하여 기존 완료 문서도 다음 출력 때 새 형식으로 다시 생성합니다.
- Power TBM 방식의 A4 생성·분할·저장·공유 흐름은 변경하지 않았습니다.

## 데이터 호환성

각 체크포인트에 다음 정보가 추가되었습니다.

```json
{
  "marker": "square | circle | dash | plain",
  "hierarchy": "parent | child | detail | standalone",
  "level": 0,
  "parentId": "상위 체크포인트 ID 또는 null",
  "sourceMarker": "■ | ○ | - | 빈 값"
}
```

- 96개 템플릿 코드 유지
- 2,578개 체크포인트 ID·문구·순서 유지
- 기존 임시저장 및 완료 문서의 응답 키 유지
- IndexedDB 데이터베이스와 문서 구조 유지
