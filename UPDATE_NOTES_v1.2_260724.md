# v1.2 iPad 작업일시 입력칸 수정 안내

## 원인

iPad Safari는 `input type="datetime-local"`의 네이티브 표시 높이를 웹 CSS보다 크게 강제하는 경우가 있습니다. 그래서 폭을 줄이는 수정만으로는 작업일시 칸이 작업장소 칸보다 계속 높게 표시됐습니다.

## 이번 수정

- 네이티브 날짜 입력기를 화면 배치에 직접 사용하지 않고 투명 선택 영역으로 변경
- 사용자에게 보이는 작업일시 칸은 42px 고정 높이로 렌더링
- 작업일시 칸을 누르면 기존 iOS 날짜·시간 선택기가 그대로 열림
- 체크리스트 작성 중에는 하단 메뉴를 숨겨 점검내용을 가리지 않게 처리
- 서비스워커 캐시 키와 파일 버전을 `20260724r3`로 갱신

## GitHub 적용
