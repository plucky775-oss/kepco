# v1.3.1 PDF 출력 수정 안내

## 해결 대상

- Android에서 PDF 오른쪽 또는 하단이 잘리는 현상
- iPhone·iPad 인쇄 미리보기에서 페이지를 직접 축소해야 하는 불편
- 화면 폭과 스크롤 위치에 따라 PDF 배치가 달라지는 현상

## 변경 내용

- 모바일 브라우저의 `window.print()` 출력 분기를 제거했습니다.
- Power TBM 회의록과 같은 이미지 기반 A4 PDF 엔진을 적용했습니다.
- 출력 전용 문서를 794px 고정 폭으로 문서 좌표 0에 렌더링합니다.
- 전체 캔버스를 A4 안쪽 196mm 폭으로 자동 축소·중앙 배치합니다.
- Android는 위아래 4mm, 그 외 기기는 6mm의 안전 여백을 적용합니다.
- 표 행, 제목, 작업절차, 보호구, 기록·보관, 서명란 경계에서 페이지를 분할합니다.
- 마지막 페이지에 소량만 남는 경우 최대 6% 범위에서 전체 폭을 조절해 페이지 수를 정리합니다.
- 서명 및 첨부사진 이미지를 최종 캔버스에 다시 합성해 이미지 로딩 타이밍에 따른 누락 가능성을 줄였습니다.
- PDF 생성 완료 후 `기기에 저장 / 카카오톡 등 공유 / PDF 미리보기` 선택창을 표시합니다.
- iPhone·iPad의 `기기에 저장`은 PDF `File`을 iOS 공유창으로 직접 전달합니다.
- Android 다운로드 파일은 서비스워커의 가상 PDF 경로를 사용해 PDF 형식과 파일명을 유지합니다.
- 기존 PDF 캐시 버전을 갱신해 과거 방식으로 만든 PDF를 자동 재생성합니다.

## 적용 후 확인

1. GitHub 또는 Vercel 재배포가 완료될 때까지 기다립니다.
2. 브라우저 페이지를 새로고침합니다.
3. 홈 화면 앱은 완전히 종료했다가 다시 실행합니다.
4. 완료 문서에서 `PDF 저장·공유`를 눌러 새 완료창이 나타나는지 확인합니다.
5. Android PDF의 좌우가 모두 보이는지, iPhone에서 인쇄 축소 없이 `파일에 저장`이 가능한지 확인합니다.

브라우저 사이트 데이터는 삭제하지 마십시오. 기존 문서가 함께 삭제될 수 있습니다.
