# 작업안전 체크리스트 독립앱 v1.5.0

GitHub Pages와 Vercel 같은 정적 호스팅에서 실행하는 독립형 PWA입니다. TBM 회의록과 연계하지 않고 작업안전 체크리스트만 단독으로 사용합니다.

## 주요 기능

- 표준공정 96종 검색·선택
- 확인 / 이상 / 해당 없음 판정
- 이상 내용, 조치사항, 현장 사진 기록
- 담당자·책임자 전자서명
- 자동 임시저장과 완료 문서 보관
- 날짜별 캘린더 조회
- A4 PDF 미리보기·기기 저장·파일 공유
- 스마트폰 홈 화면 설치와 오프라인 캐시

## v1.5.0 핵심 변경사항

원본 96페이지 PDF의 핵심 Check Point 기호와 상·하위 관계를 작성 화면과 PDF에 반영했습니다.

- `■` 사각형: 상위 점검항목
- 같은 작업절차 구간에서 사각형 뒤에 이어지는 `○` 원형: 해당 사각형의 하위 점검항목
- 다음 사각형이 나오면 이후 원형은 새 사각형의 하위 점검항목
- 사각형보다 먼저 나오거나 사각형이 없는 구간의 `○` 원형: 독립 점검항목
- `-` 대시: 직전 사각형 아래의 세부 항목
- 원본에 기호가 없는 항목: 기호 없이 독립 항목으로 유지

상위 항목은 굵은 글씨와 진한 왼쪽 선으로 표시하고, 하위 항목은 들여쓰기와 원형 기호로 구분합니다. PDF에도 같은 구조가 적용됩니다.

기존 체크포인트 ID와 저장 구조를 바꾸지 않았으므로 기존 임시저장·완료 문서의 응답은 그대로 연결됩니다. 이전에 보관된 PDF는 다음 출력 시 새 계층형 서식으로 다시 생성됩니다.

## 모바일 PDF 저장

### Android

`PDF 저장·공유` → `기기에 저장`

A4 폭에 맞춘 PDF가 기본 다운로드 위치에 저장됩니다.

### iPhone·iPad

`PDF 저장·공유` → `기기에 저장` → iOS 공유창의 `파일에 저장`

별도의 페이지 축소 조작은 필요하지 않습니다.

## 로컬 실행

압축을 푼 폴더에서 다음 명령을 실행합니다.

```bash
python -m http.server 8080
```

브라우저에서 다음 주소를 엽니다.

```text
http://localhost:8080
```

`index.html`을 파일로 직접 열지 말고 로컬 서버 또는 정적 호스팅을 사용하십시오.

## GitHub Pages 배포

1. 이 폴더의 내용 전체를 저장소 루트에 올립니다.
2. 저장소의 `Settings → Pages`로 이동합니다.
3. `Deploy from a branch`를 선택합니다.
4. 배포 브랜치와 `/ (root)`를 선택해 저장합니다.
5. 배포 완료 후 Pages 주소를 엽니다.

해시 라우팅을 사용하므로 추가 라우팅 설정은 필요하지 않습니다.

## 기존 저장소 업데이트

패치 파일을 기존 저장소 최상위 폴더에 덮어쓴 뒤 푸시합니다.

```bash
git add .
git commit -m "Distinguish parent and child checklist items"
git push
```

배포가 끝난 뒤 Safari 또는 Chrome에서 새로고침하십시오. 홈 화면에 설치한 PWA는 완전히 종료한 뒤 다시 실행해야 새 서비스워커가 반영될 수 있습니다.

브라우저의 사이트 데이터는 삭제하지 마십시오. 삭제하면 기존 임시저장·완료 문서도 함께 삭제될 수 있습니다.

## 데이터 보관

문서와 사진은 사용 중인 브라우저의 IndexedDB에 저장됩니다. 브라우저 데이터를 삭제하거나 기기를 변경하면 자동 이전되지 않습니다.
