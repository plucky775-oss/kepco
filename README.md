# 작업안전 체크리스트 독립앱 v1.3.1

Power TBM과 분리된 작업안전 체크리스트 PWA입니다. GitHub Pages와 Vercel 같은 정적 호스팅에서 실행할 수 있습니다.

## 주요 기능

- 표준공정 96종 검색·선택
- 확인 / 이상 / 해당 없음 판정
- 이상 내용, 조치사항, 현장 사진 기록
- 담당자·책임자 전자서명
- 자동 임시저장과 완료 문서 보관
- 날짜별 캘린더 조회
- PDF 미리보기·기기 저장·파일 공유
- 스마트폰 홈 화면 설치와 오프라인 캐시

## v1.3.1 PDF 출력 변경

모바일의 브라우저 인쇄 화면에 HTML을 넘기던 방식을 중단하고, Power TBM 회의록과 같은 **정적 문서 캡처 → A4 PDF 생성 → 동일 파일 저장·공유** 흐름으로 변경했습니다.

1. 현재 화면 크기나 가로 스크롤과 무관하게 출력 문서를 794px 기준 폭으로 별도 렌더링합니다.
2. 출력 문서 전체를 `html2canvas`로 고정합니다.
3. 캡처 이미지를 `jsPDF`의 A4 안쪽 폭 196mm에 자동 맞춥니다.
4. 점검표 행, 작업절차, 서명란 등의 안전 경계에서 페이지를 나눕니다.
5. 생성된 동일 PDF 파일을 미리보기·기기 저장·공유·완료문서 캐시에 재사용합니다.

이 방식으로 Android에서 오른쪽이 잘리는 현상을 막고, iPhone·iPad에서는 인쇄 미리보기에서 두 손가락으로 축소할 필요 없이 PDF 파일을 곧바로 저장·공유할 수 있습니다.

## 모바일 저장 방법

### Android

`PDF 저장·공유` → `기기에 저장`

A4 폭에 맞춘 PDF가 기본 다운로드 위치에 저장됩니다. `카카오톡 등 공유` 버튼이 지원되는 기기에서는 같은 PDF 파일을 바로 공유할 수 있습니다.

### iPhone·iPad

`PDF 저장·공유` → `기기에 저장` → iOS 공유창의 `파일에 저장`

인쇄 화면이나 페이지 축소 조작을 거치지 않습니다.

## 로컬 실행

압축을 푼 폴더에서 다음 명령을 실행합니다.

```bash
python -m http.server 8080
```

브라우저에서 다음 주소를 엽니다.

```text
http://localhost:8080
```

`index.html`을 파일로 직접 여는 대신 로컬 서버 또는 정적 호스팅을 사용해야 합니다.

## GitHub Pages 배포

1. 이 폴더의 내용 전체를 저장소 루트에 올립니다.
2. 저장소의 `Settings → Pages`로 이동합니다.
3. `Deploy from a branch`를 선택합니다.
4. 배포 브랜치와 `/ (root)`를 선택해 저장합니다.
5. 배포가 끝나면 Pages 주소를 엽니다.

해시 라우팅을 사용하므로 추가 라우팅 설정은 필요하지 않습니다. `.nojekyll` 파일도 포함되어 있습니다.

## 기존 GitHub 저장소 업데이트

패치 파일을 기존 저장소 최상위 폴더에 덮어쓴 뒤 푸시합니다.

```bash
git add .
git commit -m "Use Power TBM style A4 PDF export"
git push
```

필수 교체 파일은 다음과 같습니다.

```text
index.html
app.js
styles-checklist.css
components/research-checklist.js
sw.js
```

배포가 끝난 뒤 Safari 또는 Chrome에서 페이지를 새로고침하십시오. 홈 화면에 설치한 PWA는 앱을 완전히 종료한 뒤 다시 실행해야 새 서비스워커가 반영될 수 있습니다.

사이트 데이터는 삭제하지 마십시오. 브라우저의 사이트 데이터를 삭제하면 기존 임시저장·완료 문서도 함께 삭제될 수 있습니다.

## PDF 라이브러리

PDF 엔진은 Power TBM과 같은 `html2canvas`와 `jsPDF`를 사용합니다. 최초 실행 때 외부 라이브러리를 불러오려면 네트워크 연결이 필요합니다. 한 번 정상 로드된 파일은 서비스워커의 런타임 캐시에 저장될 수 있습니다.

## 데이터 보관 주의

현재 문서와 사진은 사용 중인 브라우저의 IndexedDB에 저장됩니다. 브라우저 데이터를 삭제하거나 기기를 변경하면 자동 이전되지 않습니다. 여러 기기 동기화와 관리자 통합조회에는 Firebase 등 서버 저장소 연동이 필요합니다.
