# 작업안전 체크리스트 독립앱 v1.8.2

GitHub Pages와 Vercel 같은 정적 호스팅에서 실행하는 독립형 PWA입니다. TBM 회의록과 연계하지 않고 작업안전 체크리스트만 단독으로 사용합니다.

## 주요 기능

- 표준공정 96종 검색·선택
- 확인 / 이상 / 해당 없음 판정
- 이상 내용, 조치사항, 현장 사진 기록
- 담당자·책임자 전자서명
- 자동 임시저장과 완료 문서 보관
- 날짜별 완료 문서 조회
- A4 PDF 미리보기·기기 저장·파일 공유
- 스마트폰 홈 화면 설치와 오프라인 캐시

## v1.8.2 변경사항

### 홈 배너 실시간 세븐세그먼트 시계

- 홈 배너 우측의 체크리스트 그림과 원형 체크 표시를 제거
- Power TBM 홈 시계의 7세그먼트 숫자 매핑과 점멸 콜론 방식을 독립앱에 맞게 적용
- 현재 시각을 `시:분` 형식으로 실시간 표시
- 시계 위에 `월 일 요일` 형식의 날짜·요일 표시
- 홈 화면을 벗어나면 타이머를 정리하고 다시 돌아오면 자동 재시작
- 320px 스마트폰부터 태블릿까지 제목과 시계가 겹치지 않도록 반응형 크기 조정

### PWA 캐시 갱신

서비스워커 캐시 버전을 `v1.8.2-r3`로 갱신했습니다. IndexedDB 저장 구조와 96종 체크리스트 데이터는 변경하지 않았습니다.

## v1.8.1 변경사항

### 체크리스트 선택 화면 간소화

- 각 체크리스트 카드에서 위험요인 미리보기 문구를 제거
- 검색창은 체크리스트 제목(작업명)만 검색하도록 범위를 제한
- 검색 문구를 `체크리스트 제목 검색`으로 명확하게 변경
- 카드의 여백과 버튼 배치를 간결하게 조정해 한 화면에 더 많은 목록이 표시되도록 개선
- 기존 대분류·중분류 필터와 절차·위험요인·점검 개수 표시는 유지

### PWA 캐시 갱신

서비스워커 캐시 버전을 `v1.8.1-r2`로 갱신했습니다. IndexedDB 저장 구조는 변경하지 않아 기존 임시저장·완료 문서를 그대로 사용할 수 있습니다.

## v1.8.0 변경사항

### 홈 화면 디자인 개편

- Power TBM 계열의 3D 체크리스트 이미지를 홈 배너와 세 가지 메뉴에 적용
- 메뉴 이모지와 설명형 보조문구를 제거하고 제목 중심으로 정리
- 칙칙한 남색 중심 화면을 밝은 블루·하늘색 그라데이션으로 변경
- 상단 버튼과 하단 메뉴를 벡터 아이콘으로 통일
- 스마트폰과 태블릿에서 카드와 이미지 크기가 자연스럽게 조정되도록 반응형 스타일 개선

### PWA 캐시 갱신

새 메뉴 이미지 네 개를 오프라인 캐시에 추가하고 서비스워커 캐시 버전을 `v1.8.0-r1`로 갱신했습니다. 기존 임시저장·완료 문서의 IndexedDB 구조는 변경하지 않았습니다.

## v1.6.0 변경사항

### 핵심 Check Point 표시

원본 PDF의 기호 모양은 유지하되 상·하위 관계를 임의로 만들지 않습니다.

- `■` 사각형 항목과 `○` 원형 항목을 같은 레벨과 같은 위치에 표시
- 사각형·원형은 카드 여백, 글자 위치, 판정 버튼 위치가 동일
- `-` 대시 항목만 원본의 세부 문장 형식을 살려 한 단계 들여쓰기
- 작성 화면과 PDF 출력에 같은 기준 적용

기존 체크포인트 ID와 순서를 유지하므로 임시저장·완료 문서의 판정 결과는 그대로 연결됩니다.

### 작업 기본정보 간소화

작업 기본정보는 다음 다섯 항목만 표시합니다.

1. 작업공종(Code)
2. 작업명
3. 공정명(과제번호)
4. 담당자
5. 책임자(PL)

작업일시, 작업장소, 회사·연구소, 부서, 작업인원, 참여 작업자, 비고 입력란을 제거했습니다. PDF 기본정보 표와 문서 목록에서도 제거된 항목을 표시하지 않습니다.

## 모바일 PDF 저장

### Android

`PDF 저장·공유` → `기기에 저장`

A4 폭에 맞춘 PDF가 기본 다운로드 위치에 저장됩니다.

### iPhone·iPad

`PDF 저장·공유` → `기기에 저장` → iOS 공유창의 `파일에 저장`

별도의 페이지 축소 조작은 필요하지 않습니다.

## 로컬 실행

압축을 푼 폴더에서 다음 명령을 실행합니다.

```bash
python -m http.server 8080
```

브라우저에서 `http://localhost:8080`을 엽니다. `index.html`을 파일로 직접 열지 말고 로컬 서버 또는 정적 호스팅을 사용하십시오.

## GitHub Pages 배포

1. 이 폴더의 내용 전체를 저장소 루트에 올립니다.
2. 저장소의 `Settings → Pages`로 이동합니다.
3. `Deploy from a branch`를 선택합니다.
4. 배포 브랜치와 `/ (root)`를 선택해 저장합니다.

해시 라우팅을 사용하므로 추가 라우팅 설정은 필요하지 않습니다.

## 기존 저장소 업데이트

패치 파일을 기존 저장소 최상위 폴더에 덮어쓴 뒤 푸시합니다.

```bash
git add .
git commit -m "Add seven-segment clock to checklist home"
git push
```

배포 완료 후 브라우저를 새로고침하십시오. 홈 화면에 설치한 PWA는 완전히 종료한 뒤 다시 실행해야 새 서비스워커가 적용될 수 있습니다.

브라우저의 사이트 데이터는 삭제하지 마십시오. 삭제하면 기존 임시저장·완료 문서도 함께 삭제될 수 있습니다.

## 데이터 보관

문서와 사진은 사용 중인 브라우저의 IndexedDB에 저장됩니다. 브라우저 데이터를 삭제하거나 기기를 변경하면 자동 이전되지 않습니다.
