# 작업안전 체크리스트 독립앱 v1.1

Power TBM과 분리된 독립 실행형 PWA입니다. GitHub Pages, Vercel 등 정적 호스팅에서 실행할 수 있습니다.

## 주요 기능

- 96종 표준공정 검색 및 선택
- 확인 / 이상 / 해당 없음 판정
- 이상 내용, 조치사항, 현장 사진 기록
- 담당자 및 책임자 전자서명
- 자동 임시저장 및 완료 문서 보관
- 날짜별 캘린더 조회
- PDF 저장·공유 및 JSON 저장
- 스마트폰 홈 화면 설치와 오프라인 캐시

## v1.1 수정 사항

- iPad·태블릿에서 작업일시 입력칸이 옆 입력칸과 겹치던 현상 수정
- `핵심 Check Point` 제목이 첫 점검항목을 가리던 고정 헤더 해제
- 하단 작성 버튼이 점검항목을 덮던 현상 수정
- 상태 선택 후 펼쳐진 내용이 하단 메뉴에 가려지지 않도록 자동 스크롤 보정
- iPhone·iPad의 PDF 백지 문제를 피하기 위해 Safari에서는 네이티브 인쇄·공유 방식 사용
- 일반 브라우저 PDF 생성 시 화면 밖 렌더링과 대형 캔버스 메모리 문제 보완
- 이전에 저장된 빈 PDF를 재사용하지 않도록 PDF 렌더 버전 갱신
- 서비스워커 캐시 버전을 올리고 배포 후 새 파일을 자동 반영하도록 개선

## 로컬 실행

압축을 푼 폴더에서 다음 명령을 실행합니다.

```bash
python -m http.server 8080
```

브라우저에서 다음 주소를 엽니다.

```text
http://localhost:8080
```

`index.html`을 파일로 직접 열기보다 로컬 서버 또는 정적 호스팅을 사용하는 것이 좋습니다.

## GitHub Pages 배포

1. 이 폴더의 **내용 전체**를 저장소 루트에 올립니다.
2. GitHub 저장소의 `Settings → Pages`로 이동합니다.
3. `Deploy from a branch`를 선택합니다.
4. 배포 브랜치와 `/ (root)`를 선택해 저장합니다.
5. 배포 완료 후 표시되는 Pages 주소를 엽니다.

해시 라우팅을 사용하므로 별도 서버 라우팅 설정은 필요하지 않습니다. `.nojekyll` 파일도 포함되어 있습니다.

## 기존 GitHub 저장소 업데이트

수정본을 기존 저장소에 덮어쓴 뒤 커밋·푸시합니다. 특히 아래 파일은 반드시 교체해야 합니다.

```text
index.html
app.css
app.js
styles-checklist.css
components/research-checklist.js
sw.js
.nojekyll
```

배포 직후 기존 화면이 계속 보이면 Safari에서 페이지를 한 번 새로고침하십시오. 홈 화면에 설치한 PWA는 앱을 완전히 닫았다가 다시 열면 새 서비스워커가 적용됩니다.

## iPhone·iPad PDF 저장

`PDF 저장·공유`를 누르면 인쇄 미리보기 화면이 열립니다. 해당 화면에서 공유 메뉴를 열어 `파일에 저장`을 선택하면 PDF로 보관할 수 있습니다. 이 방식은 iOS Safari에서 HTML 캔버스 기반 PDF가 백지로 생성되는 문제를 피합니다.

## 데이터 보관 주의

현재 작성 문서와 사진은 사용 중인 브라우저의 IndexedDB에 저장됩니다. 브라우저 데이터를 삭제하거나 기기를 변경하면 자동 이전되지 않습니다. 여러 기기 동기화와 관리자 통합조회에는 Firebase 등의 서버 저장소 연동이 필요합니다.
