# 작업안전 체크리스트 독립앱 v1.2

Power TBM과 분리된 독립 실행형 PWA입니다. GitHub Pages, Vercel 등 정적 호스팅에서 실행할 수 있습니다.

## 주요 기능

- 96종 표준공정 검색 및 선택
- 확인 / 이상 / 해당 없음 판정
- 이상 내용, 조치사항, 현장 사진 기록
- 담당자 및 책임자 전자서명
- 자동 임시저장 및 완료 문서 보관
- 날짜별 캘린더 조회
- PDF 저장·공유 및 JSON 저장
- 스마트폰 홈 화면 설치와 오프라인 캐시

## v1.2 수정 사항

- iPad Safari가 `datetime-local` 입력칸 높이를 강제로 늘리는 문제를 우회하도록 작업일시 입력 UI를 교체
- 작업일시는 항상 다른 입력칸과 동일한 42px 높이로 표시
- 표시부 전체를 누르면 기존 iOS 날짜·시간 선택기가 열리므로 날짜 변경 기능은 그대로 유지
- 체크리스트 작성 화면에서는 하단 전역 메뉴를 숨겨 점검항목이 가려지지 않도록 수정
- 서비스워커와 정적 파일 캐시 버전을 갱신
- 이전 PDF 캐시가 남지 않도록 PDF 렌더 버전을 갱신

## 로컬 실행

압축을 푼 폴더에서 다음 명령을 실행합니다.

```bash
python -m http.server 8080
```

브라우저에서 다음 주소를 엽니다.

```text
http://localhost:8080
```

`index.html`을 파일로 직접 열기보다 로컬 서버 또는 정적 호스팅을 사용하는 것이 좋습니다.

## GitHub Pages 또는 Vercel 배포

1. 이 폴더의 **내용 전체**를 저장소 루트에 올립니다.
2. 기존 저장소라면 같은 경로의 파일을 모두 덮어씁니다.
3. 커밋하고 푸시합니다.

```bash
git add .
git commit -m "Fix iPad date field layout"
git push
```

Vercel이 저장소와 연결되어 있다면 자동 재배포됩니다. GitHub Pages는 `Settings → Pages`에서 배포 브랜치와 `/ (root)`가 선택되어 있어야 합니다.

## 업데이트 후 iPad에서 꼭 할 일

배포가 끝난 뒤 Safari 탭을 닫았다가 다시 열어 주세요. 홈 화면에 설치한 앱은 앱 전환 화면에서 완전히 종료한 뒤 다시 실행해야 새 서비스워커와 CSS가 적용됩니다.

기존 작성자료를 보존하려면 Safari의 사이트 데이터는 삭제하지 마십시오. 사이트 데이터를 삭제하면 브라우저에 저장된 임시문서와 완료문서도 함께 삭제될 수 있습니다.

## iPhone·iPad PDF 저장

`PDF 저장·공유`를 누르면 인쇄 미리보기 화면이 열립니다. 해당 화면에서 공유 메뉴를 열어 `파일에 저장`을 선택하면 PDF로 보관할 수 있습니다.

## 데이터 보관 주의

현재 작성 문서와 사진은 사용 중인 브라우저의 IndexedDB에 저장됩니다. 브라우저 데이터를 삭제하거나 기기를 변경하면 자동 이전되지 않습니다. 여러 기기 동기화와 관리자 통합조회에는 Firebase 등의 서버 저장소 연동이 필요합니다.
